# Plan 05-01: VillageManager 및 NPC 서비스 인프라 구축

## 목표
정화된 NPC들의 상태를 전역적으로 관리하고, 플레이어와 적 시스템이 이를 참조하여 강화 효과를 적용받을 수 있는 기반을 마련합니다.

## 상세 작업

### 1. `VillageManager.cs` 구현 (싱글톤)
- **역할 해금**: `UnlockNPCRole(RiverNPCRole role)` 메서드를 통해 정화된 NPC 역할을 등록.
- **상태 조회**: `IsRoleUnlocked(RiverNPCRole role)`로 특정 NPC의 존재 여부 확인.
- **스탯 보너스 제공**:
    - `ShamanMaxHpBonus`: 주술사 해금 시 가산될 HP 수치.
    - `ShamanCostReduction`: 주술사 해금 시 적용될 소모량 배율 (예: 0.9 = 10% 감소).
    - `BlacksmithPurifyMargin`: 대장장이 해금 시 확장될 정화 범위 가산치.
    - `StorytellerTier0Power`: 이야기꾼 해금 시 0단계 스킬 위력 배율.

### 2. `EnemyStats.cs` 연동 수정
- `OnPurified` 가상 메서드에서 `RiverEnemyStats`가 `VillageManager.Instance.UnlockNPCRole()`을 호출하도록 연동.

### 3. 플레이어 및 적 초기화 로직 수정
- **`PlayerWaterStats`**: `Awake` 또는 `Start`에서 `VillageManager`를 참조하여 `maxCleanWater`를 보정.
- **`EnemyStats`**: `Awake`에서 `VillageManager`를 참조하여 `bonusPurificationMargin` 기본값을 보정.
- **`SkillBase`**: `ExecuteSkill` 내부의 데미지 계산 로직에서 `Stage == 0`일 때 `VillageManager` 보너스 적용.

## 검증 계획
1. **NPC 정화 테스트**: 강(River) 적 정화 시 `VillageManager`에 해당 역할이 해금되는지 로그 확인.
2. **주술사 보너스**: 주술사 해금 후 플레이어 최대 HP가 정상적으로 증가하는지 확인.
3. **대장장이 보너스**: 대장장이 해금 후 새로 생성되는 적들의 정화 범위(Sweet Spot)가 기본적으로 넓어져 있는지 확인.
4. **이야기꾼 보너스**: 0단계 공격 시 데미지 로그가 보너스 배율을 반영하는지 확인.
